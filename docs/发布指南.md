# Grok Coder 发布指南

适用于本 fork（`brucelee.grok-coder`，基于 phuryn/grok-build-vscode）。目标市场：VS Code Marketplace + Open VSX（Cursor / VSCodium 等从 Open VSX 取插件）。

## 一、一次性准备（首次发布前）

**1. Marketplace 发布者账号**

1. 用 Microsoft 账号登录 [marketplace.visualstudio.com/manage](https://marketplace.visualstudio.com/manage)，创建 Publisher，ID 必须是 `brucelee`（与 package.json 的 `publisher` 一致；如被占用，改 package.json 并全局搜索替换）。
2. 在 [dev.azure.com](https://dev.azure.com) → User settings → Personal Access Tokens 创建 PAT：Organization 选 **All accessible organizations**，Scope 选 **Marketplace → Manage**。保存好，只显示一次。
3. 本机登录：`npx @vscode/vsce login brucelee`（粘贴 PAT）。

**2. Open VSX 账号**

1. 用 GitHub 登录 [open-vsx.org](https://open-vsx.org)，签署 Publisher Agreement。
2. 个人设置页生成 Access Token。
3. 首次需创建同名 namespace：`npx ovsx create-namespace brucelee -p <token>`。

**3. GitHub 仓库**

package.json 指向 `https://github.com/lxiaolong068/grok-vscode`——发布前确保该仓库真实存在、代码已推送，否则 vsce 会警告且用户点仓库链接 404。安装 `gh` CLI 并 `gh auth login`（用于发 GitHub Release）。

**4. 品牌合规检查（fork 特有，重要）**

- ⚠️ **图标**：当前 `resources/grok-icon.png` 沿用上游，且形似 xAI 官方 Grok 标识。Marketplace 政策禁止让用户误认为是官方插件——发布前**换成自己设计的图标**，并在 displayName/描述中保留 "Not affiliated with xAI" 声明（描述里已有）。
- **命名**：displayName 用 "Grok Coder" 可以（描述兼容性用途），但不要叫 "Grok" 或 "xAI Grok"。
- **署名**：LICENSE 双版权行、README fork 声明、package.json `contributors` 已就绪，不要删。
- **遥测**：上游 Aptabase 发送点已断开（`src/sidebar.ts`）。若要恢复遥测，先在 Aptabase 注册自己的项目并替换 `src/telemetry.ts` 里的 key，绝不能用上游的 key 发布。

## 二、每次发布流程

**1. 版本与文档**

- 按语义化版本 bump `package.json` 的 `version`（当前 0.3.1）。
- `CHANGELOG.md` 顶部加新版本段（Added/Changed/Fixed，1–3 条即可）。
- 过一遍 README 的 fork 头部说明和 `docs/direct-mode.md` 是否与实际功能一致。

**2. 质量门（顺序执行，全绿才继续）**

```bash
npm test                 # 上游 640 测试为底线，全部通过
npx tsc -p . --noEmit    # 类型检查干净
npm run test:live        # ⚠️ 需要本机安装 grok CLI 且已登录订阅账号；
                         # 验证真实 ACP 链路（handshake/prompt/restore/plan mode）。
                         # 加 --quick 跳过慢的生成式测试。SKIP 不算失败，FAIL 才算。
```

直连模块（`src/direct/`）没有 live 测试脚本，发布前手动验一遍：F5 起调试宿主 → `Grok Coder: 登录 Grok 账号`（或设 API Key）→ 原生 Chat 选 Grok 4.5 问一句 → `@grok /explain` → Git 面板 ✨ 生成提交信息 → 状态栏显示正常。已知风险：部分订阅档位 OAuth API 返回 403（插件内已有提示回退 API Key，无需阻塞发布）。

**3. 打包**

```bash
npx @vscode/vsce package        # → grok-coder-<version>.vsix
```

注意：`npm run package` 的 prepackage 步骤会先删旧 vsix；上游惯例是**所有代码和文档改完后最后打包**（vsix 会把 README/CHANGELOG 打进去）。装包冒烟测试：`code --install-extension grok-coder-<version>.vsix`。

**4. Git 提交、tag 与 GitHub Release**

```bash
git add -A && git commit -m "Release v<version>"
git push origin main
git tag -a v<version> -m "Release v<version>"
git push origin v<version>
gh release create v<version> --title "Release v<version>" \
  --notes-file <(sed -n '/^## <version>/,/^## /p' CHANGELOG.md) \
  grok-coder-<version>.vsix
```

vsix 一定作为 Release 资产附上（沿用上游惯例，用户可直接下载安装）。

**5. 发布到两个市场**

```bash
npx @vscode/vsce publish                          # VS Code Marketplace
npx ovsx publish grok-coder-<version>.vsix -p <token>   # Open VSX（Cursor 用户从这装）
```

Marketplace 审核通常几分钟到几小时；首个版本可能人工审核更久。发布后在 Marketplace 页面确认 README 渲染、图标、截图正常。

**6. 发布后**

- 安装 Marketplace 线上版本再冒烟一次（偶发打包遗漏文件的问题只在线上版暴露）。
- 关注 GitHub Issues 与 Marketplace Q&A/评分。
- 上游同步：定期 `git fetch upstream && git merge`（上游改动集中在 src/ 与 media/，我们的代码都在 `src/direct/`，冲突面很小；每次合并后重跑质量门）。

## 三、发布清单（复制用）

- [ ] 图标已替换为自有设计（首次发布必须）
- [ ] version 已 bump，CHANGELOG 已更新
- [ ] `npm test` 640+ 全绿
- [ ] `tsc --noEmit` 干净
- [ ] `npm run test:live` 通过（真实 grok CLI）
- [ ] 直连模块手动冒烟（登录 / 原生 Chat / @grok / SCM 提交 / 状态栏）

- [ ] commit + tag + GitHub Release（附 vsix）

- [ ] 线上版本复检

## 四、常见问题

- **vsce 报 "Make sure to edit the README"**：README 含相对路径图片时需 `repository` 字段可访问，已配置；若截图 404 是因为 `docs/screenshots` 被 .vscodeignore 排除或仓库未推送。
- **vsce 报 personal access token 无效**：PAT 过期或 Organization 没选 "All accessible organizations"。
- **打包体积异常**：`npx vsce ls --tree` 查看打进包的文件；当前约 6.7 MB（含 media 资源与 ws 依赖），若暴涨检查 .vscodeignore。
- **EU 用户反馈用不了直连功能**：xAI API 暂未对 EU 开放（官方预计 2026-07 中旬），属上游服务限制，README 已注明。
